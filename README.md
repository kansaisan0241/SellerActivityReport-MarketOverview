# 媒介報告用 市場レポート作成ツール

HTML / CSS / JavaScriptだけで動く、媒介報告用の市場レポート作成ツールです。データベース、ログイン、サーバー保存は使いません。Vercel / Netlify / GitHub Pages に静的ファイルとして無料公開できます。

## 使い方

1. `index.html` をブラウザで開きます。
2. 入力画面で物件種別を選びます。
3. 物件データをCSVまたはTSV形式で貼り付けます。
4. 表示したいブロックにチェックを入れます。
5. 「市場レポートを自動生成」を押します。
6. レポート画面で比較表、地図、分布図、価格変更判定を確認します。
7. 「印刷」からPDF保存または紙出力します。

地図は Leaflet + OpenStreetMap、グラフは Chart.js を使用しています。緯度・経度がない物件は地図表示の対象外です。

## JSONエクスポート方法

1. 入力画面で物件データを読み込みます。
2. JSONインポート／エクスポート画面を開きます。
3. 「JSONを書き出し」を押します。
4. テキスト欄に現在データのJSONが表示されます。
5. 「ファイル保存」を押すと `.json` ファイルとして保存できます。

エクスポートしたJSONは、次回の媒介報告時に前回データとして比較できます。

## JSONインポート方法

1. 前回エクスポートしたJSONを開きます。
2. JSONインポート／エクスポート画面の「前回JSONインポート」に貼り付けます。
3. 「前回データとして読み込む」を押します。
4. 今回データと照合し、新規物件、掲載終了・成約候補物件、価格変更物件を自動判定します。

同一物件判定は、次のいずれかが一致した場合に行います。

- 所在地＋土地面積
- 所在地＋専有面積
- 所在地＋建物面積

同一物件と判定された場合、初回登録日は前回データを引き継ぎます。

## GitHub Pagesで無料公開する方法

1. GitHubで新しいリポジトリを作成します。
2. `index.html`, `style.css`, `app.js`, `README.md` をリポジトリ直下にアップロードします。
3. リポジトリの `Settings` を開きます。
4. `Pages` を選択します。
5. `Build and deployment` の `Source` を `Deploy from a branch` にします。
6. Branchを `main`、フォルダを `/root` にして保存します。
7. 表示された GitHub Pages のURLにアクセスします。

VercelやNetlifyでも、同じファイル一式を静的サイトとしてデプロイすれば利用できます。

## データベースを使わない説明

このツールは完全フロントエンドのみで動作します。入力された物件データはブラウザ上のJavaScriptで処理され、サーバーやデータベースには保存されません。

前回比較が必要な場合は、利用者自身がJSONを書き出して保管し、次回そのJSONをインポートします。そのため、無料の静的ホスティングだけで運用できます。
